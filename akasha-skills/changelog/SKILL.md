# Akasha Changelog Generator

This skill transforms git commits and system updates into readable, user-friendly changelogs that reflect the evolution of the Akasha ecosystem.

## When to Use This Skill
- After completing a new feature or bug fix.
- Before preparing a new release or update to the Akasha Core.
- Summarizing the progress of the "Council of Librarians."

## Instructions
1. **Commit Analysis**: Analyze git commit history since the last release tag.
2. **Category Grouping**: Group changes into categories: (e.g., "Council Updates," "Core Intelligence," "UI/UX Enhancements," "Bug Fixes").
3. **Sovereign Tone**: Maintain a formal yet inspiring tone ("The Librarians have synthesized...").
4. **Breaking Changes**: Clearly highlight any changes that affect existing data storage or database schemas (Alembic).

## Examples
- "Generate a changelog for the recent updates to the 'Oracle' and 'Scribe' librarians."
- "Summarize the changes in the 'Chronos Engine' for the v1.2.0 release."
